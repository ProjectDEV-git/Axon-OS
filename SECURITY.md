# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in Axon OS, please **do not** open a public issue. Instead, please report it responsibly to the maintainers.

### Reporting Process

1. **Email:** Send a detailed report to [security@axon-os.local] (or contact via GitHub)
2. **Include:**
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fix (if you have one)

3. **Response Time:** We aim to acknowledge receipt within 48 hours and provide a status update within 1 week

### Security Update Process

1. We will confirm the vulnerability
2. Develop and test a fix in a private branch
3. Release a security patch (minor or patch version bump)
4. Publicly disclose the vulnerability after the patch is released
5. Credit the reporter (unless they prefer anonymity)

## Dependency Security

- We use **Dependabot** to automatically check for vulnerable dependencies
- Security updates are prioritized and merged quickly
- All dependencies are regularly audited

## Best Practices

When using Axon OS:
- Keep your system and dependencies up to date
- Report security issues responsibly
- Review code and permissions before running untrusted code
- Run Axon OS from a trusted source (GitHub)
